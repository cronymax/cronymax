## ADDED Requirements

### Requirement: Explain/Fix/Retry actions on ShellBlock spawn a topic thread

Each completed `ShellBlock` SHALL show an action bar with **Explain**, **Fix**,
and **Retry** buttons. Clicking any of these SHALL create a `Thread` object
attached to that block and expand it inline in the timeline.

#### Scenario: Fix action creates and expands a thread inline

- **WHEN** the user clicks [Fix] on a failed ShellBlock
- **THEN** a `Thread` with `action: "fix"` is created and attached to the block
- **AND** the thread is expanded inline below the shell block's output
- **AND** the thread's `systemContext` is set to the block's command and output
- **AND** the first assistant message in the thread is automatically generated by `agent.run` with the system context

#### Scenario: Thread reply sends within thread context

- **WHEN** the user submits a message via the inline thread reply composer
- **THEN** the message is sent to `agent.run` with `history` = thread blocks only
- **AND** the system prompt includes the thread's `systemContext`
- **AND** a new `ConversationBlock` is added to `thread.blocks`

### Requirement: Thread can be collapsed or switched to full-screen view

A thread SHALL support two presentation modes: **inline expanded** (default)
and **thread view** (full-screen timeline replacement). Both modes SHALL share
the same `Thread` data.

#### Scenario: Collapse inline thread

- **WHEN** the user clicks [Collapse ↑] on an expanded inline thread
- **THEN** the thread collapses to a one-line summary card showing action, message count, and agent name
- **AND** the thread remains accessible via [View thread ⇄] on the summary card

#### Scenario: Switch to thread view

- **WHEN** the user clicks [View thread ⇄] or [Switch to thread view ⇄]
- **THEN** `activeView` changes to `{kind:"thread", threadId, parentBlockId}`
- **AND** the timeline renders only the thread's `ConversationBlock[]`
- **AND** a pinned context card showing the parent shell block's command + output appears at the top
- **AND** a breadcrumb `← <chatName>` is shown in the header

#### Scenario: Return to main timeline from thread view

- **WHEN** the user clicks the breadcrumb `← <chatName>` in thread view
- **THEN** `activeView` changes to `{kind:"main"}`
- **AND** the full block timeline is restored
- **AND** the shell block that owns the thread shows a collapsed summary card
